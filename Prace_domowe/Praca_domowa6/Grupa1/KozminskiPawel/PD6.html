Poszukiwanie najlepszej liczby klastrów dla algorytmu K Means¶

Poszukiwania bd wsparte uycie dwch metryk oceny jakoci metod klasteryzujcych:

  • Wspczynnik Silhouette - obliczany przy pomocy dwch wartoci dla kadej obserwacji: $a$ - rednia odlego midzy innymi punktami z tego samego klastra oraz $b$ - rednia odlego do innych punktw z innego klastra. Nastpnie obliczana jest warto: $\frac{a-b}{max(a, b)}$. Funkcja silhouette_score() zwraca urednion warto dla wszystkich punktw. Silhouette osiga wartoci od -1 do 1, przy czym ujemna warto wspczynnika dla pewnej obserwacji oznacza, e zostaa jej przyporzdkowana niewaciwa grupa. Oczywicie im wikszy wspczynnik osiga wynik, tym lepszy by proces klasteryzacji,
  • Indeks Daviesa-Bouldina - miara obliczajca rednie 'podobiestwo' pomidzy klastrami, ktre jest okrelane jako iloczyn odlegoci wewntrz klastrw i odlegoci midzy klastrami. Wobec tego, rne klastry powinno uzyskiwa nisze wyniki - im nisza warto indeksu, tym lepiej zadziaaa klasteryzacja.

def two_metrics_plot(scores, clusters_range, metricname1, metricname2, x_label = 'Liczba klastrw'):
    fig, ax = plt.subplots()
    ax.plot(clusters_range, scores[0], 's-', color = 'orange')
    ax.set_ylabel(metricname1, color = 'orange')
    ax.tick_params(axis = 'y', labelcolor = 'orange')
    ax.set_xlabel(x_label)
    ax2 = ax.twinx()
    ax2.plot(clusters_range, scores[1], 's-', color = 'blue')
    ax2.set_ylabel(metricname2, color = 'blue')
    ax2.tick_params(axis = 'y', labelcolor = 'blue')
    plt.xticks(clusters_range)
    plt.title('Warto metryk klasteryzacji')

def model_scatter_3d(data, model, **kwargs):
    Model = model(**kwargs)
    labels = Model.fit_predict(data)
    fig = go.Figure(data = [go.Scatter3d(x = data.iloc[:,0], y = data.iloc[:,1], z = data.iloc[:,2],
                              mode = 'markers', marker = dict(size = 1.5, color = labels))])
    n_cluster = len(np.unique(labels))
    to_title = '.'
    if -1 in labels:
        n_cluster-=1
        to_title += " Wystpiy 'szumy'."
    fig.update_layout(title = model.__name__ + ', liczba klastrw: ' + str(n_cluster) + to_title)
    return fig

DBSCAN - drugi sprawdzany algorytm klasteryzujący¶

Korzystajc z DBSCAN kluczowym jest ustalenie dwch parametrw: eps - maksymalna odlego midzy dwoma obserwacjami, aby byo rozwaane ich ssiedztwo oraz min_samples - minimalna liczba obserwacji w ssiedztwie aby punkt by traktowany jako centrum. Korzystajc z informacji zawartych w artykule Jrg Sander, Martin Ester, Hans-Peter Kriegel, and Xiaowei Xu. 1998. Density-based clustering in spatial databases: The algorithm GDBSCAN and its applications. Data Mining and Knowledge Discovery2, 2 (1998), 169194. (artyku patny - cytowanie za Schubert, Erich & Sander, Jrg & Ester, Martin & Kriegel, Hans & Xu, Xiaowei. (2017). DBSCAN revisited, revisited: Why and how you should (still) use DBSCAN. ) zdecydowaem si na zastosowanie arbitralnego sposobu ustawienia tych parametrw:

  • min_samples = $2 *$ wymiar danych
  • eps = odlego, wzgldem ktrej wikszo spord $k$-tych ssiadw dla poszczeglnych obserwacji jest bliej. $k$ jest obliczane przy pomocy formuy $2 * $ wymiar danych $- 1$

Ponadto, niektre obserwacje mog by traktowane jako wartoci odstajce - w takim przypadku zostanie wystosowany odpowiedni komunikat.

def calculate_kn_distance(X,k):
    kn_distance = []
    for i in range(len(X)):
        eucl_dist = []
        for j in range(len(X)):
            eucl_dist.append(euclidean(X.loc[i,:], X.loc[j,:]))
        eucl_dist.sort()
        kn_distance.append(eucl_dist[k])
    return kn_distance
